Configuración y puesta en funcionamiento
Instalación en infraestructura de la Entidad con documentación de requisitos previos
Instalación en infraestructura del cliente
MonitorInn [DDBB] se instala completamente en la infraestructura de la Entidad con un único comando. El instalador desatendido (install.sh / install.ps1) configura automáticamente todos los componentes incluyendo base de datos interna, certificado SSL y variables de entorno. No hay dependencias de nube.
install.sh · install.ps1 · docker-compose.yml
✓ Cumple
Requisitos mínimos documentados (SO, IPs, puertos, instancias)
Se entrega el documento técnico de pre-requisitos (prereq-monitorinn.docx) con especificaciones de hardware, sistema operativo, puertos de red (TCP 443, 1433, 389/636, 25/587), configuración de doble interfaz de red, usuarios de Active Directory requeridos y script SQL de configuración de instancias.
prereq-monitorinn.docx · scripts/setup_instance.sql
✓ Cumple
Implementación limpia y segura
Arquitectura agentless: ningún software se instala en los servidores SQL Server. La cuenta de monitoreo es de solo lectura (VIEW SERVER STATE). Doble interfaz de red aísla el tráfico de usuarios del tráfico hacia bases de datos. Todos los secretos se generan automáticamente durante la instalación.
app/collectors/base.py · nginx.conf · install.sh

Características mínimas de la implementación
Capacidades técnicas requeridas una vez implementada la solución
Monitoreo de rendimiento del motor
Monitoreo de comportamiento CPU
Métricas de CPU del motor SQL Server capturadas en tiempo real: SQL Compilations/sec, Batch Requests/sec y consumo de CPU por query (avg_cpu_ms) desde Query Store. Polled cada 60 segundos, visualizado en Dashboard con semáforo de umbral.
sys.dm_os_performance_counters · sys.query_store_runtime_stats
✓ Cumple
Monitoreo de memoria
Page Life Expectancy (PLE) y Buffer Cache Hit Ratio monitoreados en tiempo real. PLE mide cuánto tiempo las páginas permanecen en memoria (>300s saludable). Cache Hit Ratio indica el porcentaje de lecturas servidas desde memoria (>95% ideal). Alertas automáticas al cruzar umbrales críticos.
sys.dm_os_performance_counters (Buffer Manager)
✓ Cumple
Monitoreo de I/O
Lecturas lógicas promedio por query (avg_logical_io_reads) disponibles en la pantalla Top Queries, filtrable por base de datos y período. Lock Waits/sec monitorea contención de bloqueos que generan I/O de espera. Las queries activas muestran logical_reads en tiempo real.
sys.query_store_runtime_stats · sys.dm_exec_requests
✓ Cumple
Top queries y consultas de alto consumo
Pantalla dedicada "Top Queries" lee directamente del Query Store nativo de SQL Server 2022. Ordena por duración promedio, CPU promedio, lecturas lógicas o número de ejecuciones. Filtros por base de datos y período (30min, 1h, 6h, 24h, rango personalizado). Actualización automática cada 30 segundos.
sys.query_store_runtime_stats · sys.query_store_plan · sys.query_store_query_text
✓ Cumple
Plan cache analysis
Cada query en la pantalla Top Queries permite visualizar el plan de ejecución XML directamente desde el Query Store. El plan se renderiza en un modal con scroll. En la pantalla de Dashboard, las sesiones activas exponen el plan de ejecución en vivo mediante sys.dm_exec_query_statistics_xml.
sys.query_store_plan · sys.dm_exec_query_statistics_xml
✓ Cumple
Bloqueos y sesiones activas
Tabla de sesiones activas en tiempo real en el Dashboard (actualización 5s): session_id, estado, tipo de espera (wait_type), tiempo transcurrido y SQL. Deadlocks capturados automáticamente desde la sesión system_health de XEvents de SQL Server. Alerta automática cuando el número de sesiones bloqueadas supera el umbral configurado.
sys.dm_exec_requests · sys.dm_xe_session_targets (system_health)
✓ Cumple
Monitoreo de índices fragmentados
Colector dedicado que consulta sys.dm_db_index_physical_stats para detectar índices con fragmentación superior al 30% (warning) y 60% (critical). Vista específica con listado por base de datos, porcentaje de fragmentación, tipo de operación recomendada (REBUILD vs REORGANIZE) y alertas automáticas al superar los umbrales configurados.
sys.dm_db_index_physical_stats · app/collectors/index_health.py
✓ Cumple
Alertas e historial
Alertas basadas en umbrales configurables
Motor de alertas evalúa métricas tras cada ciclo de polling (60s). Los umbrales son configurables por instancia o globalmente desde la interfaz web (Config → Alertas). Soporta nivel warning y critical por métrica. Supresión automática de alertas duplicadas en ventanas de 30 minutos.
app/alerting/engine.py · alert_thresholds (PostgreSQL)
✓ Cumple
Histórico de alertas
Todas las alertas disparadas se registran en la tabla alerts_history con: instancia, métrica, valor que cruzó el umbral, nivel de severidad (warning/critical) y timestamp. La pantalla Centro de Alertas muestra el historial con filtros por severidad e instancia.
alerts_history (PostgreSQL) · app/api/routes/alerts.py
✓ Cumple
Notificaciones por correo electrónico
Envío automático de email al momento de dispararse una alerta, vía SMTP interno de la Entidad (configurable: host, puerto, remitente, destinatarios). Compatible con relay SMTP en puerto 25 o 587. No requiere acceso a internet — usa el servidor de correo interno.
app/alerting/smtp.py · aiosmtplib
✓ Cumple
Seguridad y acceso
Autenticación integrada con Active Directory
Login con credenciales de dominio AD. MonitorInn [DDBB] verifica la pertenencia al grupo de seguridad monitorinn antes de validar la contraseña. Tokens JWT (15min) con refresh en cookie httpOnly (7 días). Ninguna contraseña se almacena en MonitorInn [DDBB] — toda la gestión de usuarios es responsabilidad del AD de la Entidad.
app/auth/ldap.py · app/auth/jwt.py · python-ldap
✓ Cumple
Eventos y observabilidad del motor
Monitoreo de eventos críticos del motor SQL
Tres fuentes de eventos del motor: (1) sesión XEvents propia MonitorInn_Trace captura queries con duración >1s en buffer en memoria, (2) sesión system_health nativa de SQL Server 2022 captura deadlocks y errores severos, (3) Error Log del motor (sys.fn_read_error_log) para mensajes de error y advertencias del motor.
sys.dm_xe_session_targets · sys.fn_read_error_log · app/collectors/xevents.py
✓ Cumple
Reportes y análisis
Reportes ejecutivos predefinidos
Reporte PDF ejecutivo exportable desde la pantalla Reportes: incluye resumen de KPIs del período, tendencias de PLE, Cache Hit Ratio y Batch Requests/sec, y top 5 queries más costosas del período. El rango de fechas es seleccionable. Generado con WeasyPrint (sin dependencias de nube).
app/api/routes/reports.py · app/templates/report.html · WeasyPrint
✓ Cumple
Dashboards personalizables
MonitorInn [DDBB] incluye un Dashboard con los 6 KPIs estándar del motor SQL Server, gráficas de alertas e instancias, y tabla de sesiones activas. El selector de instancia y el período de análisis son configurables por el usuario. La personalización avanzada de layout por usuario (drag & drop de widgets) está en desarrollo.
src/pages/Dashboard/index.tsx · src/components/MetricCard.tsx
~ Parcial
Análisis de tendencias de capacidad
MonitorInn [DDBB] captura snapshots horarios de métricas clave y los visualiza como gráficas de tendencia histórica en la pantalla Reportes (PLE, Cache Hit Ratio, Batch Requests/sec). El análisis predictivo con proyecciones a 30/60/90 días y alertas anticipadas de saturación está en desarrollo.
metric_snapshots (PostgreSQL) · src/pages/Reports/index.tsx
~ Parcial
Histórico de cambios de configuración
Módulo de auditoría que detecta y registra cambios en parámetros del motor SQL Server mediante sys.configurations y el Error Log. Almacena el historial de valores anteriores/actuales con timestamp, y genera alertas automáticas al detectar cambios no programados en parámetros críticos (memoria máxima, grado de paralelismo, autenticación, etc.).
sys.configurations · sys.fn_read_error_log · app/collectors/config_changes.py
